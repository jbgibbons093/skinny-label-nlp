About & Methods

This tool visualizes the semantic relationship between individual patent claims, Orange Book use codes, and approved label indications for FDA-approved drugs. Claim-indication similarity is now the headline measure of direct patent-label alignment, while scope gap remains a secondary use-code-derived summary.

+ Show methods

Three-Way Semantic Similarity

All text (patent claims, use code descriptions, label indications) is embedded with S-PubMedBert-MS-MARCO (768-dim, biomedical domain). Cosine similarity in this shared space measures semantic closeness. Three pairwise similarities are computed:
  • Claim ↔ Use Code: Does the patent's language match the use code description?
  • Indication ↔ Use Code: Does the drug label's approved indication match the use code?
  • Scope Gap: = Claim-UC similarity minus Indication-UC similarity. Positive = patent claims reach beyond label.

Scope Gap

At the edge level: scope_gap = cosine_sim(claim, UC) - cosine_sim(indication, UC)
Each claim-UC pair has its own scope gap. Positive = that claim semantically exceeds the label for that use code. The NDA-level scope gap is a summary statistic (mean of per-edge gaps). This is an exploratory measure; formal validation (AUC against manual mappings) is pending.

Similarity Floor

Biomedical embeddings have a floor at ~0.82 cosine similarity — even unrelated texts score this high. The meaningful range is 0.82–1.0. Our threshold of 0.88 corresponds to the 10th percentile of true indication↔use code matches. Edges below 0.88 are considered "unmatched."

Claim Selection

Only independent method-of-use claims are included (53,081 claims from 5,872 patents). Dependent claims (175,580) are excluded — they are legally subordinate and rarely the best semantic match for a use code. Including them slightly attenuates the scope gap (mean 0.0086 vs 0.0097 independent-only) because they narrow toward specific indications that also appear in the label.

Data Sources

  • 2,927 NDAs (1,966 with use codes, 961 control)
  • 228,661 patent method-of-use claims
  • 17,761 parsed label indications
  • 4,421 FDA use code descriptions
  • Orange Book snapshots 1988–2024

Network Data

343,712 claim-UC edges (independent claims with cosine ≥ 0.88) across 1,913 NDAs. Each edge includes a pre-computed scope gap (g = claim_sim - indication_sim). Each use code includes its indication-UC similarity (iu) and UC-level scope gap (ug = max_claim - iu).

Limitations

  • Validation against manual claim-UC mappings is pending (Phase 0)
  • Compressed similarity range (0.82–1.0) limits interpretability
  • 234 of 1,966 treatment NDAs lack scope gap data (missing labels or parsing failures)
  • 42 NDAs lack claim-level network data

Model Architecture & Pipeline

Three Text Registers

Patent claims, Orange Book use-code descriptions, and approved label indications remain separate semantic objects throughout the interface.

Embedding Layer

S-PubMedBert-MS-MARCO embeds all text types in one 768-dimensional space so claim, use-code, and indication comparisons stay directly comparable.

Similarity Engine

The site now foregrounds Claim↔Indication as the direct patent-label measure, while still exposing Claim↔UC, Indication↔UC, and derived scope gap for context.

Website Architecture

site_data.json powers fast overview panels. Claim-level networks are lazy-loaded per NDA from networks/ and cached in-session.

Performance Controls

  • Per-drug lazy loading
  • Hard SVG caps for dense views
  • Reduced Sankey on heavy drugs
  • Persistent filters with timeline playback

Validation Status

Claim-indication is the headline interface metric, but all semantic measures remain exploratory. Manual mapping validation is still pending, so the site emphasizes comparison, inspection, and separability rather than hard classification.

How To Use The Explorer

1. Pick a drug 2. Click a year in a chart 3. Filter claims, patents, and UCs 4. Compare Network, Heatmap, and Sankey

The timeline and filters are designed to work together: once you isolate a set of claims or use codes, the same slice follows you through time and across visual views.

Interpretive Guidance

A high claim-indication score means patent claims semantically track approved label indications closely. Scope gap remains useful, but it is treated here as a secondary, use-code-mediated signal rather than the main headline.

Legal & Policy Context

Skinny labeling turns on whether patented uses can be separated cleanly from retained label language. This interface helps trace where patent claims, use codes, and indications appear aligned, overlapping, or difficult to disentangle.

Current Export Limits

  • No curated therapeutic class variable is present in the website JSON export.
  • Directory filtering therefore uses ingredient and keyword matching as a class proxy.
  • Large claim-level views may render reduced subsets for speed and stability.
  • Some treatment NDAs lack scope-gap output because labels are missing or parsing failed.
